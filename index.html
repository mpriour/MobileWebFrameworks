Mobile WebApp Development Frameworks

Cutting Through The Hype

  • Matt Priour (OpenGeo)

Buzzwords

Search for these terms and get all kinds of results

  • webapp framework
  • html5 webapp development
  • javascript webapp development framework
  • etc...

What do we want

Required

  • Javascript/HTML5/CSS3 Web Apps - Not Native
  • Desktop compatible with modern browers
  • Single page style sites
  • Useful and Extendable Widgets Library
  • Develop using JS & HTML
  • Free & Open-Source

What do we want

Preferred

  • MVC-ish Structure
  • Evented data stores
  • Data-linked Templates & Widgets
  • Easily themed & customized
  • Project generator & packager

What we don't want

  • Mobile Microframeworks
    • Zepto.js, xui - No mobile ui. Simply cross-device DOM & Events
  • Titanium
    • Must be compiled to native apps
  • Spine or Backbone alone
    • Just an MVC framework, no mobile ui. Still dependent on HTML id
  • Frameworks that generate JS/HTML from other languages
    • GWT (Java), RhoMobile (Ruby), Coffescript, etc
  • Kendo UI and other such frameworks
    • Not free for open source. Closed to semi-closed source

No Thanks!

jQueryMobile
http://jquerymobile.com

  • Features
    • Clean separation of views (declarative HTML) & code
    • Good set of UI widgets & page navigation system
    • Easy to theme via CSS.
      Theme Roller (http://jquerymobile.com/themeroller/) also available
    • Extremely broad device support (Android 2.1+, 3.x, 4.x; iOS 3+; IE 8+; FF, Chrome, Safari)
  • Missing
    • Data driven templating
    • Evented data store
    • Need to include
      Stapes (http://hay.github.com/stapes/) & Tempo (git://github.com/twigkit/tempo.git)
      to really have a more full-featured framework

qooxdoo
http://qooxdoo.org

  • Features
    • All pages defined in JS
    • Large set of UI widgets
    • Themed / Customized via LESS
    • Mostly automatic widget to data model connection & update
    • Python based SDK for project generation & packaging
  • Missing
    • MVC file structure - each "page" is a class containing:
      a view definition, controllers, and a data model instance
      Similar to defining panels in ExtJs 3
    • Good learning resources - Very limited examples & tutorials

The M-Project
http://the-m-project.org

  • Features
    • MVC Framework. All real logic is defined in controllers
      • All views defined in JS & event references by string
    • Wraps Jquery Mobile
      • Extremely broad device support & good UI widgets
    • Node.js based build & dev server tools
      • Uses phoneGap / Cordova for native device building
      • Very deep set of project setup, build, deploy & native compile commands
  • Missing
    • Documentation - Docs & examples are better than qooxdoo but nothing like jQueryMobile or Sencha Touch
      • You can probably do a lot with this framework, but the examples & docs are not deep enough to really know
    • Configuration & Styling flexibility of JQM
    • Easier inclusion of external libraries. They must be copied into the project
    • Easier remote data source configuration in models / stores

Sencha Touch 2
http://docs.sencha.com/touch/2-0/

  • Features
    • MVC Framework. All real logic is defined in controllers
      • All views defined in JS & event references by string
    • Good UI Widget Set
    • Uses similar (but NOT same) syntax to ExtJS4
    • Evented data stores that easy to configure for remote data sources
  • Missing
    • Broad device support
      • No offical support for Android 3.x, IE 8 or lower, limited Blackberry support
    • Better styling documentation.
        • Styling is done via LESS but only a video tutorial explains it
